# Festivus Academy — Unreal shell

This project is an asset-light Unreal Engine 5.8 C++ shell for the Developer Academy.

## What already works

- A procedural four-door academy atrium with no marketplace asset dependency.
- A durable `/Game/Maps/AcademyAtrium` startup map generated by the checked-in authoring script.
- Camera, lighting, fog, room labels, and a cinematic dark-academy composition.
- An asset-free UMG hub created in C++, with truthful locked-room states.
- A cinematic portal-focus/return camera state machine with reduced-motion support.
- A nonce-gated WKWebView bridge that reveals a room only after React hydration.
- A covered timeout state that keeps unfinished pages hidden while allowing late recovery or a safe return.
- Per-entry browser teardown so room sockets, media, and timers cannot leak into the hub.
- Embedded access to the original Festivus teaser.
- Embedded access to every web curriculum room.
- Runtime catalog discovery through `GET /api/academy/rooms`.
- An AI-first mission workbench that teaches specify → generate → review → validate.
- Offline fallback: the built-in room buttons still open known routes.

## Run locally

1. From the repository root, run `pnpm dev:all`.
2. Ensure Unreal Engine 5.8 is installed with C++ support.
3. Open `unreal/FestivusAcademy/FestivusAcademy.uproject`.
4. Allow Unreal to compile the `FestivusAcademy` module.
5. Press Play. The academy reads `http://localhost:3000` by default.

To recreate or normalize the startup map after native world changes, run:

```bash
"/Users/Shared/Epic Games/UE_5.8/Engine/Binaries/Mac/UnrealEditor-Cmd" \
  "$PWD/unreal/FestivusAcademy/FestivusAcademy.uproject" \
  -run=PythonScript \
  -script="$PWD/unreal/FestivusAcademy/Tools/create_academy_map.py" \
  -Unattended -NoSplash -NoSound -NoP4
```

The script preserves exactly one `AcademyWorldDirector` and saves the map
idempotently. Pass `-AcademyCapture -RenderOffscreen` to a `-game` launch to
write a deterministic scene frame to
`Saved/Screenshots/Mac/AcademyAtrium.png`, then exit automatically. Runtime logs
also confirm that the native HUD constructs, mounts, and reaches the live catalog.

For a deterministic room lifecycle check, add `-AcademyAutoEnter -AcademyAutoReturn`
to a windowed `-game` launch. Add `-AcademyAutoLoops=2` to prove the native
browser can be destroyed and recreated across repeated room visits. The log must
show portal entry, hydration, `about:blank`, browser resource release, and restored
atrium input in that order. Add `-AcademyReducedMotion` to validate the
no-camera-motion path from the first atrium frame.

To point the shell at production, change `ApiBaseUrl` in `Config/DefaultGame.ini`:

```ini
[Academy]
ApiBaseUrl="https://festivus-game.vercel.app"
```

## Why the UI is code-generated

Binary Unreal assets are fragile in source-only automation and cannot be meaningfully
reviewed in Git. The first vertical slice generates its atrium and UMG shell from C++.
Once the editor build is green, artists can replace procedural geometry with Nanite
environments and Blueprint subclasses without changing the API or curriculum model.

## Validation

`pnpm unreal:validate` checks project/module/plugin/config wiring without an engine.
A real Unreal compile is still required before calling the native client shippable.
